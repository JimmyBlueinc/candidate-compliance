## ğŸ“ Where to Upload (Step by Step)

### FRONTEND FILES (Your Website)

**Location:** The EMPTY folder you see (usually called `public_html/` or `www/` or `httpdocs/`)

1. Open the File Manager in your hosting control panel
2. You should see a folder like:
   - `public_html/` (most common)
   - `www/`
   - `httpdocs/`
   - Or just your domain name folder
3. **This folder is currently EMPTY - that's correct!**
4. Upload ALL files from the `frontend-dist/` folder into this empty folder
5. After upload, you should see:
   - index.html
   - .htaccess
   - assets/ (folder)
   - vite.svg

### BACKEND FILES (Your API)

**Location:** Create a NEW folder OUTSIDE the website folder

1. In File Manager, go UP one level from `public_html/`
2. You'll see your home directory (usually `/home/username/` or similar)
3. Create a NEW folder called `backend`
4. Upload ALL files from the `backend/` folder into this new `backend/` folder
5. After upload, you should see inside `backend/`:
   - app/
   - public/
   - storage/
   - vendor/
   - .env.example
   - artisan
   - composer.json
   - ... (many other files)

## ğŸ” How to Find Your Website Folder

If you're not sure which folder is your website:

1. **Look for these names:**
   - `public_html/` (most common in cPanel)
   - `www/`
   - `httpdocs/` (Plesk)
   - `htdocs/`
   - Your domain name (e.g., `yourdomain.com/`)

2. **Check your hosting documentation:**
   - Most hosts have a "File Manager" guide
   - Look for "Where to upload website files"

3. **Contact support:**
   - Ask: "Where should I upload my website files?"
   - They'll tell you the exact folder name

## ğŸš€ Next Steps After Upload

1. **Create `.env` file:**
   - In `backend/` folder, copy `.env.example` to `.env`
   - Edit `.env` with your database credentials

2. **Set up database:**
   - Go to MySQL Databases in cPanel
   - Create database and user
   - Update `.env` with credentials

3. **Make backend accessible:**
   - Option A: Create subdomain `api.yourdomain.com` pointing to `backend/public/`
   - Option B: Copy `backend/public/` contents to `public_html/api/`

4. **Run setup commands** (if terminal/SSH available):
   ```
   cd backend
   php artisan key:generate
   php artisan migrate --force
   php artisan storage:link
   php artisan config:cache
   ```
